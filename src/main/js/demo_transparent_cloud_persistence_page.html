Summary

This demo allows to make posts to a status or message board. A text typed in the input box above will appear as item in a list below the input box upon clicking the [Post] button.

All posts are persisted using the Appjangle cloud storage. You can click on the button Open in New Window to open a new window, in which the posts you made are loaded from the Appjangle cloud storage.

Note

Cloud Persistence
In this demo, all posts are persisted in the Appjangle cloud. The source code, however, is almost identical to the instant data demo, which worked with a local database. The high responsiveness of the application is maintained without any need for additional caching code, since data is kept in a local embedded database, which is transparently synchronized with the Appjangle cloud.
No Collaboration
Opening the posts in a new window allows to view them but not make new posts. Check the state synchronization and fine-grained sharing demos to see how the post can easily be shared and edited in multiple locations.

Usage of Appjangle

Appjangle provides a local database to store the posted messages as well as the synchronization engine to persist messages in the Appjangle cloud.

The JavaScript code of this demo is almost identical to the JavaScript code used for the first demo (instant data), which used a local embedded database.

The following source code extracts from demo_transparent_cloud_persistence.js illustrate the key differences to the first demo (demo_instant_data.js):

An embedded db server as in the instant data demo is not required. In any case the session created with Nextweb.createSession() continues to maintain a local copy of all accessed nodes. 

// BEFORE: server = Nextweb.startServer(12322);
session = Nextweb.createSession();

We call the seed() operation and not the operation seed('local'). This will lead to a call being sent to the Appjangle cloud, when the seed node is requested. The returned seed node will then have a globally unique URI, which we can use to share the appended posts with any REST capable application. 

// BEFORE: posts = session.seed("local");
posts = session.seed();
