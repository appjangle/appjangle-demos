Summary

This demo allows to make posts to a status or message board. A text typed in the input box above will appear as item in a list below the input box upon clicking the [Post] button.

All posts are persisted to the Appjangle cloud storage. You can click on the button [Open in New Window] to try out loading the posts from the cloud.

Note

Cloud Persistence
In this demo, all posts are persisted in the Appjangle cloud. The source code, however, is almost identical to the instant data demo, which worked with a local database. The high responsiveness of the application is maintained without any need for additional caching code, since data is kept in a local embedded database, which is transparently synchronized with the Appjangle cloud.
No Collaboration
Opening the posts in a new window allows to view them but not make new posts. Check the state synchronization and fine-grained sharing demos to see how the post can easily be shared and edited in multiple locations.

Usage of Appjangle

Appjangle provides the local database containing the posted messages. The Nextweb open source API is used to interact with the Appjangle embedded database in this example.

Following the annotated source code of this demo illustrating the use of Appjangle.

demo_instant_data.js

An embedded db server is started and a client session is created.

// local db server to manage data
server = Nextweb.startServer(12322);
session = Nextweb.createSession();

The seed operation is used to seed a new root node in the Appjangle database. This root node works as the 'container' for all application data. All other data is appended as direct and indirect child nodes to this root node.
