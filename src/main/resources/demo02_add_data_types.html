Prepare Data Type

In the model, we use the data type aPost. In order to use this data type in our application, we first need to create it. Luckily, this is very easy, since the property pattern allows any arbitrary node to serve as a type node (see Nextweb documentation) and creating nodes is very easy in Appjangle.

The following script creates a new data type node 'aPost' and prints out the address for this node. This address is all we need to use the newly created node as globally available, reusable and documented data type!

We can also save the 'secret' for the node in order to be able to edit and modify our data type later on.

var session = Nextweb.createSession();

var typesRoot = session.seed();

var aPost = typesRoot.append("A post on a message board, wall or chat room.", "./aPost");

session.commit().get(function(success) {
    console.log("URI: "+aPost.get().uri());
	console.log("Secret: "+aPost.get().secret());
    session.close().get(function(success) { });
});

Note: you can click on the link to the generated node to explore its automatically generated web representation. By adding to the end of the URI .value.xml or .value.json, you can also easily see the other rendered representations of the node.
