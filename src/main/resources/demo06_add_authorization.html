Demo 06: Add Authorization

In demo 05, multiple users access a shared node (posts), which enables them to see each other's posts. For this, each client needs to be passed a URL for the posts node as well as a valid secret (check demo05_spawn.js for details), which enables to read and write this node and all its children.

This is not desirable, since it enables any user to freely edit posts made by other users.

In this demo we therefore add a simple security model to our application: every user can see posts made by the other users but not edit them.

To illustrate this model, we allow users to highlight posts they have made and deem of particular importance by clicking one the icon. However, they can only highlight the posts they have made themselves! Check the live demo below and find the key changes made to demo 05 explained below.
