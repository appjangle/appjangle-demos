Conclusion

The previous 10 demos showcase various capabilities of the Appjangle platform. In particular, we have explored the iterative development of a simple application 'The Wall of the Wise'.

While this application is by no means complete as yet, we can say with confidence that we have assembled a fairly substantial prototype thus far.

In this conclusion, we would like to highlighted a number of benefits from using the Appjangle platform, which are illustrated by the demos. Please find these benefits listed below.

No Database Configuration and Server Code

While our application persists data in the cloud and supports the synchronization between multiple users, we did not have to write a single line of classical 'server' code.

Smooth Cross-Platform Development

In demo 09, we could easily implement a Java client for our previously JavaScript-only application.

For this, we could ignore the fact that we are accessing data created in a JavaScript environment. All data type mapping/data conversion/etc is silently taken care of in the background and we can focus on building the application logic we are about.

Concise Database Logic

Although our clients are not mere 'UI wrappers' of the required functionality but embed the entire app features, the logic to interact with the database is concise an limited to a couple of lines of code.

For instance, the most advanced version of the application to make posts to the wall demo06_add_authorization.js contains only 30 lines of code, which contain references to the Nextweb API.

Easy Data Structure Extensability and Evolveability

Starting with a very simple model developed in demo 01, we have evolved the data model of our application in small incremental steps:

  • Demo 02 added a data type 'aPost' to identify posts.
  • Demo 05 added the data types 'anAvatar' and 'aUserName'.
  • Demo 06 added a data type 'aRemarkablePost'.

All these changes did not affect the 'backward compatibility' of the data stored by our server. For instance, a wall created with demo 06 could without problems be loaded by the client in demo 05.

High Modularity

We have created the more complex demos out of a number loosely coupled modules. Demo 07, for instance, consists of the following modules:

Module Script Description
Spawn demo07_spawn.js To start clients apps and moderator app.
Login demo07_login.js To register or sign in using email address and password.
Wall demo06_add_authorization.js To make posts to the Wall of the Wise.
Moderator demo06_moderation_app.js To highlight posts as remarkable.

As can be seen, from the used JavaScript files, this demo 07 reuses modules from demo 06. Moreover, most of the modules used in demo 07 are in turn used by demo 08 as well.

The only dependency these demos share is the application data stored in the Appjangle cloud. There are no direct source code dependencies between the modules, which allows to easily compose and recombine the modules to react to evolving requirements.

Rapid Application Evolution

From the very simple text-only prototype developed in demo 01, we were able to rapidly extend the functionality of our application by drawing on the ready-made building blocks, the Appjangle platform provided.

For instance, the following key steps in the development of our application usually require significant resources to be implemented:

  • From demo 03 to demo 04, we changed only a couple of lines of code, yet our application was transformed from a JavaScript-only client-based application to an application which persists its state in a centralized database.
  • From demo 04 to demo 05, we allowed our application to be 'collaborative' in that multiple users are enabled to see each others actions.
  • From demo 06 to demo 07, we added a 'traditional' user registration and login dialog. We could accomplish this by reusing the Appjangle user management facilities and could leave the logic of our main application unaltered.

These steps are not only notable due to velocity in which they could be accomplished but also because they solve 'hard' problems, which can easily 'make or break' an application. For instance, turning a client-only prototype into an application, which draws its data from a central database and does so with acceptable performance, can take up the majority of development time. In Appjangle, we get this feature virtually for free.

The Platform for Smarter Apps

While using Appjangle provided us with numerous benefits listed above, the most significant advantages can be drawn by the unique combination of all these benefits.

While not needing to implement a server, building easily for multiple-platforms, expressing database logic concisely in few lines of code, and utilizing the powerful building blocks of the Appjangle platform allow us to build application rapidly, the easy and flexible extension of data structures and intrinsic high modularity give even quickly built application sustainable value; in that they are easy to maintain, recombine and extend.
