Demo 08: Add Security

While in demo 07 we have added a way for users to authenticate with our application, it is so far not 'very' secure. There are in fact ways beyond listing in which our 'Wall of the Wise' could be compromised by Mallroys and other evil-meaning folk.

In this demo, we add one simple measure to improve the security of our application. In particular, we are interested in guarding the access token used by the moderator client to mark posts as remarkable. See the live demo below, which allows secure access to the moderator application!

Key Changes

The only change we had to make was to change the way moderator clients are started up.

In particular, we have changed the URL pointing to the moderator client from:

http://appjangle.com/showcases/moderatorClient

to:

https://appjangle.com/showcases/moderatorClient

Now, the parameter we pass to the moderator client containing the access token for the moderator node, as well as all communication between the moderator client and the Appjangle cloud will be encrypted through the SSL protocol.
